# 🚀 Setup Replit — Mpanolontsaina IA (Frontend React)

## 1. Fichiers à uploader dans Replit
Crée un Repl **React (Vite)**, puis uploade/copie ces fichiers dans la structure indiquée :

| Fichier fourni | Destination dans le Repl |
|---|---|
| `colors.js` | `src/theme/colors.js` |
| `translations.js` | `src/i18n/translations.js` |
| `client.js` | `src/api/client.js` |
| `01-DESIGN-SYSTEM.md` | `docs/01-DESIGN-SYSTEM.md` (référence, pas exécuté) |
| `02-INTEGRATION-GUIDE.md` | `docs/02-INTEGRATION-GUIDE.md` (référence) |

Uploade aussi tes documents originaux dans `docs/` pour que Replit AI (Agent/Assistant) garde le contexte complet :
`01_architecture_et_bdd.md`, `02_contrats_api_auth_users.md`, `03_contrats_api_chat.md`, `04_guide_implementation_vertex_ai.md`, `05_guide_switch_provider_mistral_vertex.md`, `ETAT_DU_PROJET.md`, `README.md`.

## 2. Variable d'environnement à créer sur Replit
Dans **Secrets** (ou `.env` à la racine) :
```
VITE_API_BASE_URL=http://api.mpanolontsaina-ia.duckdns.org
```

## 3. Prompt à coller dans Replit Agent / Assistant

```
Construis le frontend React (Vite) complet de "Mpanolontsaina IA", un assistant
juridique intelligent trilingue (malagasy/français) pour Madagascar, en te basant
STRICTEMENT sur les fichiers du dossier docs/ :

- docs/01-DESIGN-SYSTEM.md → identité visuelle, composants, responsive à respecter
- docs/02-INTEGRATION-GUIDE.md → comment connecter l'API
- docs/02_contrats_api_auth_users.md et docs/03_contrats_api_chat.md → contrats API exacts
- src/theme/colors.js → SEULE source de couleurs, ne jamais coder un hex en dur
- src/i18n/translations.js → SEULE source de textes FR/MG, ne jamais coder un texte en dur
- src/api/client.js → client API déjà prêt, à utiliser tel quel

L'API backend externe est déjà en ligne à cette adresse :
http://api.mpanolontsaina-ia.duckdns.org (base /api/v1, voir client.js)
Ne PAS créer de backend, ne PAS mocker les réponses : consommer cette API réelle.

Pages à créer : Home, Chat (accessible aux visiteurs sans compte), Login, Register,
History (conversations, auth requise), Account (profil + suppression de compte).

Exigences non négociables :
1. Toutes les couleurs viennent de src/theme/colors.js (variables, jamais de hex en dur)
2. Tous les textes viennent de src/i18n/translations.js via le hook useI18n()/t()
3. Sélecteur de langue MG/FR visible en permanence dans le header
4. Design responsive mobile/tablette/desktop selon les breakpoints du design system
5. Afficher systématiquement les sources juridiques (sources[]) sous chaque réponse IA
6. Gérer les erreurs API (429 rate limit visiteur, 401 non authentifié, 500) avec les
   messages traduits correspondants (chat.rateLimited, chat.errorGeneric, etc.)
7. Stocker access_token/refresh_token/session_id dans localStorage comme prévu dans client.js

Commence par créer la structure de dossiers, le thème Tailwind branché sur colors.js,
puis le layout général (Header + Sidebar + zone de contenu), puis la page Chat en priorité.
```

## 4. Ordre d'exécution conseillé une fois le Repl créé
1. Coller le prompt ci-dessus dans l'Agent Replit.
2. Vérifier que `tailwind.config.js` importe bien `colors.js`.
3. Lancer `npm run dev` et tester `/chat` en mode visiteur (aucune connexion requise).
4. Tester `register` → `login` → `history` pour valider le flux authentifié.
5. Si erreur "mixed content" (HTTPS Replit → HTTP API) : activer le proxy Vite décrit dans `02-INTEGRATION-GUIDE.md §2`.
